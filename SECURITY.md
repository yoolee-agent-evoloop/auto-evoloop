# Security Policy

## Reporting a Vulnerability or Sensitive Data Exposure

Do not report suspected vulnerabilities, leaked credentials, private endpoints, or sensitive data in public issues.

Please use GitHub Private Vulnerability Reporting:

https://github.com/yoolee-agent-evoloop/auto-evoloop/security/advisories/new

When reporting, include:

- A concise description of the issue.
- The affected file, commit, or URL.
- The minimum evidence needed to reproduce or verify the issue.
- Whether you believe credentials, private data, or user data may be exposed.

## Data Handling Rule

The public repository must not contain:

- Real user or customer conversations.
- Health, financial, legal, debt, payment, contract, or other sensitive business data.
- Production traces, logs, screenshots, exports, or analytics payloads.
- API keys, tokens, cookies, passwords, private keys, or `.env` files.
- Internal service URLs, private package registries, or production database endpoints.

Use synthetic examples only.
