# S3 Reflection: Plan Quality

Use after `auto-fix-planner` produces `fix_plan.md`.

## Checks

- Each fix maps back to one or more S2 findings.
- The plan starts with the smallest useful change.
- Validation commands and success criteria are explicit.
- Rollback notes exist for risky changes.
- Human approval state is visible.

## Stop Conditions

- A fix has no evidence link.
- The plan changes unrelated behavior.
- Validation cannot detect the target improvement.
- The plan lacks approval.

## Recommendation Format

```markdown
### S3 Reflection
- Recommendation:
- Fixes approved:
- Fixes needing revision:
- Validation gap:
- Required next action:
```
