## 本 stage 角色
M issue + 历史 baseline + 架构状态 → fix_plan（含 tradeoff、threshold、rollback path）

## 输入 artifacts
- 当前 round 的 fix_plan.md（approved 状态）
- M issue 列表（来自 S2）
- 历史 baseline / noise floor / 历史方差
- 跨轮 plan 历史（同 issue 在前面 round 的 plan 决策）

## 4 类根因在 S3 的典型 surface

> 指引而非清单——SKILL.md Step 2 的 4 类原理才是查的依据；这里只列"本 stage
> 最常出现在哪些 surface"，帮你定位。具体洞察由模型从原理推导出来。

- **A 测量信号** → 输入 issue 来源 / baseline 数值是否可信 / noise floor 是否引用 / 历史方差是否量化 / 跨轮 baseline 口径是否一致
- **B 决策规则** → 优先级策略 / threshold 公式（是否相对 noise 而非绝对值）/ rollback 触发条件 / tradeoff 评估方法 / HITL 升降级规则
- **C 跨层信号（含延迟 loop）** → fix_plan 的 expected_tradeoffs / interaction_risks / rollback_triggers 是否结构化字段而非 prose；S4 的 D1 算法能否消费这些字段；plan 输出有无 verification_signal 让 S4 出 outcome 后回流验证
- **D 成本曲线（跨轮）** → 同 issue 跨轮 plan 是否冲突（学习率震荡）/ 累积 tradeoff 总和是否单调向坏 / 跨轮 threshold 漂移 / 同方向重复尝试次数

## 数值一致性必查
- threshold 是否引用 noise floor / 历史方差
- expected delta vs noise std 比较
- rollback trigger 阈值是否可观测
- tradeoff 量化分（+X% main metric / -Y% 副指标）

## v1 → v1+ 升级提示
跑过 ≥ 3 轮 S3 反思后，若发现某些 surface 反复命中或反复无产出，更新本文件。
