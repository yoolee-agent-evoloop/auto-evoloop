# S3 Reflection: Plan Quality

## Purpose

Check whether `fix_plan.md` is executable and reviewable.

## Inputs

- Draft or approved fix plan.
- S2 feedback and case reports.

## Outputs

- S3 reflection note.
- Recommendation to approve, revise, reject, or return to analysis.

## Procedure

1. Map each fix to evidence.
2. Check that each fix is focused.
3. Confirm validation commands and success criteria.
4. Confirm rollback notes.
5. Confirm approval state.

## Human Gate

Stop when fixes lack evidence, validation cannot detect the target behavior, or
approval is missing.

## Failure Modes

- Unrelated changes are bundled.
- A fix has no evidence link.
- The plan is executable only with hidden private context.
