# S2 Reflection: Attribution Quality

## Purpose

Check whether case analysis is strong enough for planning.

## Inputs

- `case_report.md`.
- `feedback.json`.
- Source trace or runner output references.

## Outputs

- S2 reflection note.
- Recommendation to proceed, revise S2, or return to S1.

## Procedure

1. Confirm expected and observed behavior are both stated.
2. Verify each finding has evidence.
3. Check alternative hypotheses.
4. Confirm the repair hint is no broader than the evidence.
5. Mark weak evidence as `needs_more_evidence`.

## Human Gate

Stop when attribution is speculative, evidence is missing, or multiple cases are
conflated.

## Failure Modes

- The report invents evidence.
- The repair hint jumps to a broad rewrite.
- The finding requires private context not present in artifacts.
